# Desktop rice — alacritty + awesome

Minimal Arch + Awesome WM + Alacritty setup. Tiling-focused, riced lightly.

## What to install on a fresh Arch box

```sh
sudo pacman -S --needed \
    alacritty awesome zsh \
    rofi picom feh \
    network-manager-applet blueman polkit-gnome \
    pamixer playerctl brightnessctl flameshot \
    ttf-jetbrains-mono-nerd
```

Then install oh-my-zsh:

```sh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"
```

### What each package does
- **alacritty** — terminal
- **awesome** — window manager
- **zsh + oh-my-zsh** — shell (theme: `essembeh`, plugin: `git`)
- **rofi** — application launcher (Mod+d)
- **picom** — compositor; required for alacritty's `opacity = 0.85`
- **feh** — wallpaper (owns `~/.fehbg`)
- **nm-applet / blueman / polkit-gnome** — systray applets for network, bluetooth, GUI auth prompts
- **pamixer** — volume + the wibar volume widget
- **playerctl** — media-key control of running players
- **brightnessctl** — laptop backlight
- **flameshot** — region/full screenshot tool
- **ttf-jetbrains-mono-nerd** — font (with icon glyphs used in the wibar)

## Wallpaper

Owned by `~/.fehbg`, generated by running feh once:

```sh
feh --bg-fill /path/to/wallpaper.jpg
```

Awesome's autostart re-runs `~/.fehbg` each session.

## Alacritty bindings

- **Shift+Enter** — literal newline (instead of executing the line)

Everything else is alacritty defaults (Ctrl+Shift+C / Ctrl+Shift+V for clipboard, etc.).

## Awesome keybinds

`Mod` = Super (Windows key). `Mod+s` opens the keybinds help popup inside awesome itself.

### Tags (workspaces)
| Bind | Action |
|---|---|
| `Mod+1..9` | view tag N |
| `Mod+Shift+1..9` | move focused window to tag N |

### Focus / arrangement
| Bind | Action |
|---|---|
| `Mod+j` / `Mod+k` | focus next / previous window |
| `Mod+Shift+j` / `Mod+Shift+k` | swap window with next / previous |
| `Mod+h` / `Mod+l` | shrink / grow master pane |

### Window state
| Bind | Action |
|---|---|
| `Mod+f` | toggle fullscreen |
| `Mod+Ctrl+Space` | toggle floating for current window |
| `Mod+Space` | toggle layout (tile ↔ floating) |
| `Mod+Shift+c` | close window |

### Launchers
| Bind | Action |
|---|---|
| `Mod+Return` | open alacritty |
| `Mod+d` | rofi (apps) |

### WM control
| Bind | Action |
|---|---|
| `Mod+s` | show keybinds help popup |
| `Mod+Ctrl+r` | reload rc.lua |
| `Mod+Shift+q` | quit awesome (no confirm — careful) |

### Hardware keys
- `XF86AudioRaiseVolume` / `Lower` / `Mute` / `MicMute` — pamixer
- `XF86AudioPlay` / `Next` / `Prev` — playerctl
- `XF86MonBrightnessUp` / `Down` — brightnessctl
- `Print` — flameshot region; `Mod+Print` — flameshot full screen

### Mouse on a window
- `Left-click on edge` — drag-resize (4px invisible strip on left/right/bottom)
- `Mod+Left-drag` — move window
- `Mod+Right-drag` — resize window

## Notes

- **No top titlebar.** Window title shows in the wibar tasklist.
- **Floating rules** are minimal: `pinentry`, `Arandr`, role `pop-up` (e.g. Chrome detached devtools). Anything else tiles — add to `awful.rules.rules` in `rc.lua` if you need exceptions.
- **Autostart**: picom, nm-applet, blueman-applet, polkit-gnome, `~/.fehbg`. Each guarded by `pgrep` so `Mod+Ctrl+R` (reload) doesn't duplicate processes.
- **PRIME**: an `xrandr --setprovideroutputsource modesetting NVIDIA-0` line runs on startup. It's a no-op on non-NVIDIA-Optimus boxes — leave it.
- **zsh startup**: `ZSH_DISABLE_COMPFIX="true"` in `.zshrc` to skip oh-my-zsh's per-startup completion-dir audit (saves ~50ms).
